## Lab 1
### 1. Тема
__Контест система__. Система позволяет проверять студентов/учеников/сотрудников на соответствие нужным знаниям в различных направлениях с помощью: открытых/закрытых вопросов, запуска написанного кода на тестах, дописывания кода в существующий модуль.

__NOTE__: Закрытые вопросы должны поддерживать разные типы данных, т.е. отображение в вариантах ответа картинок, текста, формул и т.д. и т.п.

### 2. BA (Business Analys)

#### Яндекс.Контест
Яндекс.Контест — платформа для проведения соревнований. Преимущественно используется для проведения чемпионатов по спортивному программированию, а также имеет возможность проводить тестирование с открытыми и закрытыми вопросами. Условия можно писать в форматах Markdown, Latex, Html.

Целевая аудитория:
- Преподаватели
- Компании-организаторы
- Олимпиадники
- Студенты
- Школьники

Преимущества:
- Ролевая система доступов к контестам
- Облачное решение
- Разные методы контроля знаний
- Многообразие форматов условий
- Современный и удобный интерфейс
- Возможность обратной связи с организаторами контеста
- Возможность публикации организаторами обьявлений
- Возможность ограничения контеста по времени (продолжительности и/или началу и концу)
- Антиплагиат кода
- Ограничение используемых компиляторов на соревновании

Недостатки:
- Нерасширяемый функционал
- При большом количестве одновременных участников необходимо оповестить платформу для поддержания работоспособности
- Необходимость запрашивать доступ к полному функционалу организатора на платформе

#### Moodle
Moodle — open-source образовательная платформа. Преимущественно используется для обучения студентов/школьников. Имеет возможность проведения тестов с вопросами открытого и закрытого типов, и публикации теоретических материалов в форматах текста и видео. Условия можно писать в форматах latex, html. Также имеет возможность дополнять базовый функционал с помощью сторонних(в том числе самописных) модулей.

ЦА:
- Преподаватели
- Студенты

Преимущества:
- Open-source
- Ролевая система доступа к курсам
- Расширяемость функционала
- Хранение теоретических материалов
- Методы контроля знаний: открытые/закрытые вопросы
- Публикация обьявлений по курсам
- Ограничение по времени выполнения тестов

Недостатки:
- Open-source
- Самостоятельная установка/настройка/обновление системы на собственном сервере
- Несовременный интерфейс
- Отсутствие возможности тестирования кода "из коробки"
- PHP для сторонних модулей

#### Google Forms
Google Forms — облачные сервисы для проведения опросов/викторин. Позволяет создавть и аннализировать пользовательские опросы/викторины, а также конвертировать их результаты в формат Excel для дополнительной обработки/анализа.

ЦА:
- Аналитики
- Преподаватели

Преимущества:
- Облачное решение
- Методы контроля знаний: открытые/закрытые вопросы
- Ролевая система доступа к форме
- Возможность ручного закрытия форм
- Возможность быстрого создания опросов/викторин
- Возможность удобного перехода к анализу/обработке результатов с помощью таблиц Excel

Недостатки:
- Отсутствие возможности тестирования кода
- Нерасширяемый функционал
- Функционал сконцентрирован только на проведении опросов/викторин для сбора данных

#### Solve

Solve — специализированные open-sourse платформы для проведения соревнований по спортивному программированию. Активно используеться в образовательных целх в БГУИР/БГУ.

ЦА:
- Организаторы олимпиад
- Олимпиадники

Преимущества:
- open-sourse
- Ролевая система доступов к контестам
- Тестирование кода программ "из коробки"
- Возможность обратной связи с организаторами контеста
- Возможность публикации организаторами обьявлений
- Возможность ограничения контеста по времени (продолжительности и/или началу и концу)
- Антиплагиат кода
- быстрая настройка системы

Недостатки:
- Сложно расширяемый функционал (монолитная архитектура)
- Отсутсвие формата открытых/закрытых вопросов
- Самостоятельная установка/настройка/обновление системы на собственном сервере

#### Codeforces
Codeforces — платформы специализирующаяся на проведении соревнований по спортивному программированию. Платформа предоставляет собственные и партнерские соревнования, а также дает возможность проведения закрытых тренировочных соревнований. Поддерживается ИТМО.

ЦА:
- Олимпиадники
- Компании-организаторы

Преимущества:
- Ролевая система доступов к контестам
- Тестирование кода программ
- Возможность обратной связи с организаторами контеста
- Возможность публикации организаторами обьявлений
- Возможность ограничения контеста по времени (продолжительности и/или началу и концу)
- Удобная система составления задач (Polygon)
- Рейтинговая система участников на основании их результатов участия в соревнованиях
- Внедрение инновационных форматов задач

Недостатки:
- Не расширяемый функционал
- Отсутсвие формата открытых/закрытых вопросов
- Возможность создания открытого соревнований только при наличии определенного рейтинга либо при согласовании с админитрацией платформы
- Отсутсвует ограничение используемых компиляторов для тренировочных соревнований
- Задачи становяться общедоступными после проведения открытого соревнования

#### Stepik

Stepik — облачная образовательная платформа. Преимущественно используется для обучения студентов/школьников. Имеет возможность проведения тестов с вопросами открытого и закрытого типов, и публикации теоретических материалов в форматах текста и видео. Условия можно писать в форматах latex.

ЦА:
- Обучающиеся
- Преподаватели

Преимущества:
- Простое и уобное создание публичных курсов
- Ролевая система доступа к курсам
- Возможность совмещения теоритических материалов вместе с контролем знаний
- Хранение теоретических материалов
- Методы контроля знаний: открытые/закрытые вопросы и тестирование кода
- Публикация обьявлений по курсам
- Возможность создания платных курсов

Недостатки:
- Облачный сервис
- Не расширяемый функционал
- Отсутсвие антиплагиата
- Тестирование кода не являеться приоритетным форматом контроля знаний
